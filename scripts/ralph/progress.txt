## Codebase Patterns
- LangChain packages are hoisted to root node_modules (monorepo); imports resolve from there
- apps/web/.gitignore has `.env*` — must add `!.env.example` to track example env files
- Use `createReactAgent` from `@langchain/langgraph/prebuilt` (not from `langchain` package directly)
- `createReactAgent` param is `llm` (not `model`), accepts `checkpointer` directly
- Supabase server client uses cookies() — do NOT use inside LangChain tools (use JWT-initialized client instead)
- Use `tool()` from `@langchain/core/tools` (not `DynamicStructuredTool`) — it passes `RunnableConfig` as 2nd arg to func
- Access userId/supabaseClient via `config.configurable` in tool functions
- check_ins table column is `check_in_type` (not `type`), no `reflection` column exists
- Shared package import is `@swe-mentor/shared` (not `@repo/shared`); `Profile` type is pre-exported
- `createReactAgent` supports `prompt` (async function), `preModelHook`, `postModelHook` — no middleware array
- `check_and_increment_chat_usage` RPC returns array — access as `data[0].allowed`, `data[0].remaining`
- Use `m.getType()` (not `_getType()`) on BaseMessage to check message type ("human", "ai", "tool")
- For JWT-based auth in API routes, use `createClient` from `@supabase/supabase-js` (not the cookie-based `@supabase/ssr` server client)
- Use `crypto.randomUUID()` instead of `uuid` package — available in Node.js, no extra dependency
- `createReactAgent` accepts `prompt` (async function) and `preModelHook` — pass these in `createMentorAgent` factory
- `agent.stream()` with `streamMode: "messages"` yields tuples — check `msg instanceof AIMessageChunk && typeof msg.content === "string"` before emitting
