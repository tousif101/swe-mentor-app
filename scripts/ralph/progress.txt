## Codebase Patterns
- Validation schemas live in `apps/mobile/src/utils/validation.ts`, tests in `__tests__/validation.test.ts`
- Use `ROLE_CONFIG` and `ROLES_ORDERED` from `lib/roleMapping.ts` as single source of truth for roles ‚Äî never duplicate role lists
- Vitest is the test runner for mobile app: `npx vitest run` from `apps/mobile/`
- Pre-existing typecheck errors in App.tsx and e2e/ are unrelated ‚Äî only check for errors in modified files
- Zod enum requires a non-empty tuple type ‚Äî use `as unknown as readonly [string, ...string[]]` when converting arrays
- Seed scripts go in `scripts/` at project root; use service role key from `npx supabase status --output json`
- `career_matrices` and `level_definitions` tables: use `mapToLevelDefinition()` pattern for consistent level insertion
- Custom hooks live in `apps/mobile/src/hooks/` with barrel export in `index.ts`; `useCompanyMatch` is the shared hook for company autocomplete
- Settings screens have mixed styling: CareerGoalScreen uses `StyleSheet.create()`, EditProfileScreen uses NativeWind `className` ‚Äî match the existing approach per file
