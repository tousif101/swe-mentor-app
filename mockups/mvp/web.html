Your AI Mentor

Helping you grow from IC3 → IC4

I saw you're working on a feature flags RFC. That's a great initiative for IC4-level visibility! How can I help you with it?

I'm not sure how to structure the RFC. What should I include to make it convincing?

Great question! For an effective RFC that demonstrates IC4-level thinking, I'd recommend this structure:

  1. Problem Statement - Tie to recent incidents (like the retro discussion)
  2. Proposed Solution - Compare 2-3 options with trade-offs
  3. Implementation Plan - Phased rollout approach
  4. Cost Analysis - Include both tool cost and eng time
  5. Success Metrics - How will you measure impact?

Would you like me to help you draft any of these sections?

Yes, let's start with the problem statement. How do I make it compelling?

Based on your journal entry about the retro, here's a draft problem statement:

"Our current deployment process requires full rollbacks when issues are detected, resulting in X hours of downtime per incident. In Q4, we had 3 such incidents affecting Y users. Feature flags would enable instant feature disabling without deployment, reducing MTTR from hours to seconds."

The key is connecting the problem to business impact. Do you have access to incident data to fill in the specifics?
